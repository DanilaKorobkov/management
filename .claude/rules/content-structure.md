### Ключевые соглашения

- **`processes/`** — контентные markdown-файлы, синхронизируются в Confluence (пространство MIS)
- **`charts/`** — SQL-запросы для Redash, организованы по теме (tech-debt, и т.д.)
- **`*.private.*` / `*.private/`** — не публикуются в Confluence. Убери `private` из имени для публикации
- **`_index.md`** — overview папки, при sync становится телом родительской страницы в CF
- **`datasets/`** — описания витрин DWH. Читать перед генерацией SQL-запросов. Новую витрину добавлять как `datasets/<table_name>.md` + маппинг в `sql-conventions.md`

## Соглашение о `_index.md`

Каждая папка с контентом имеет файл `_index.md` — он описывает саму папку: краткий overview содержимого + ссылки на разделы дочерних страниц.

Правила:
- `_index.md` при синхронизации становится телом **родительской страницы** папки в CF, а не отдельной дочерней страницей
- Вложенность папок = вложенность страниц в CF
- Ссылки в `_index.md` — локальные (`[текст](path/to/file.md#anchor)`), резолвятся в CF URL при sync
- Это соглашение действует для **всех** папок репозитория, не только `processes/`

Структура репозитория — source of truth. Confluence (и возможные будущие цели экспорта) зеркалят её 1:1.

## Форматирование Markdown-таблиц

Все таблицы в `.md`-файлах должны быть выровнены по столбцам для удобства чтения в исходном коде:
- Ячейки дополняются пробелами до ширины самого длинного значения в столбце
- Разделительная строка (`|---|`) повторяет ширину заголовков
- Выравнивание применяется при создании и редактировании таблиц

## SQL-запросы

При генерации SQL-запросов к DWH:
- **Всегда** читай `datasets/sql-conventions.md` перед написанием запроса
- **Всегда** используй описания витрин из `datasets/` для понимания схемы и связей
- **Engine:** ClickHouse (не Vertica) — используй ClickHouse-синтаксис (`toDate()`, `dateTrunc()`, `numbers()`, `countIf()`)
- **JOIN:** ClickHouse не поддерживает неравенства в `JOIN ON` — используй `CROSS JOIN` + `countIf()`/`WHERE`
- **Комментарий:** вверху каждого запроса — блок `/* ... */` с описанием логики и параметров на русском языке
